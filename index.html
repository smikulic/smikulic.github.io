If you have 5 different stylesheets, how would you best integrate them into the site?

From a performance perspective, a monolithic stylesheet is best because the request number is a bigger performance concern than the size of one. This is why I would concatenate them into one bigger stylesheet file and then minimize it using build tools so I can keep my development files separate and happy in their structure.

Can you describe the difference between progressive enhancement and graceful degradation?

Sure! progressive enhancement is mostly encouraged by mobile first approach for developing responsive websites and would like you to display all the important informational sections on the smallest desired screen and then proportionally scale it up so it fits bigger screens and changes the shape to fit them.

Graceful degradation keeps your focus on desktop screens and encourages you to remove information/features which wouldn't make sense to keep on smaller screens.

For example. Graceful degradation would like you to remove huge slider which uses some HD images and heavy weight scripts to run on mobile phones which wouldn't be able to run those scripts or would have no marketing impact when presenting those images - so it would be replaced by simple text. Using progressive enhancement method you could either start building slider with a much more standardized scripts or go with the decision that the slider won't even be necessary on desktop screen but the text should just appear larger.

What is Flash of Unstyled Content? How do you avoid FOUC?

When browsers encounter a stylesheet loading directive, they can either stall parsing and wait for the stylesheet to load, keep going, or stall on demand. Safari currently uses the "keep going" approach. This triggers an FOUC when a script tries to access a property that the browser doesn't know the value of because the CSS hasn't loaded yet. The effect of it havign a website look like it's unstyled for a few seconds.

Simple method for minimizing FOUC is by hiding all or part of the web page until all styles and JavaScript are finished by applying a class name "js" as the selector that hides all content within a container that has an id="fouc".

HTML Questions:

What does a doctype do?

Doctype describes the HTML that will be used in our page and it’s placed at the top of the HTML markup. Browsers also use the doctype to determine how to render a page and whether to use quirks mode or not.

What's the difference between standards mode and quirks mode?

Standards mode support website behaviours described by HTML and CSS specification, while quirks more layout emulates non-standard behaviour in older browsers.

What's the difference between HTML and XHTML?

XHTML is HTML that follows the XML rules along with HTML tags.

Are there any problems with serving pages as application/xhtml+xml?

Only if preferred User agent content serving policy is different it is considered safer to serve preferred content as well. Issues might occur while using older Internet Explorer since anything sent as application/xhtml+xml is considered as downloadable content.

What are the advantages/disadvantages of using CSS preprocessors?

Advantages are clearly code maintainability and code reduction because of ability to write variables and mixins, other than that I provides easier CSS file structure and within the CSS selectors itself it provides clear encapsulation with it's nesting capabilities.

As a downside this could also lead to huge CSS files being generated. Which will happen if developers are not paying enough attention to how they utilize preprocessor language, such as nesting selector too deep. And of course added complexity to the project because of package that will handle preprocessing would be added to the downside list.
